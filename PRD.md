## 1. 产品概述

### 1.1 产品定位
专为医美机构咨询师打造的专属 AI 助理（Copilot + Agent），核心能力链路：

**语音 MP3 解析 → 客户多维标签自动生成 → 客户资产管理 → 跟进策略自动生成 → 跟进任务自动执行**

覆盖"管理端 + 业务端"双层场景，UI 采用**未来科技感设计语言**，强化 AI 产品的智能感知与品牌差异化。

### 1.2 核心价值
| 角色 | 价值点 |
|---|---|
| 管理方（院方） | 掌握全局业务数据；管理咨询师团队；统一 SOP 标准；客户资产沉淀在机构而非个人 |
| 咨询师 | 降低记忆成本；随时查看客户标签与跟进记录；AI 自动生成策略与话术；提升转化与复购 |

### 1.3 商业逻辑
AI Agent 的核心价值在于**堵住客户转化漏斗的每个漏点**：

- **管理层面**：统一 SOP → 提效降错、快速培养新人、防止离职带走客户数据
- **业务层面**：AI 辅助咨询 → 提升转化率、复购率、客单价与满意度
- **收入层面**（MVP 后验证确定）：
  - 方案 A：按咨询师席位收费（SaaS 年费）
  - 方案 B：按客户档案数阶梯收费
  - 方案 C：基础席位费 + 增值用量包（语音解析时长 / 自动执行次数）
  - **推荐 A + C 混合模式**：席位费保底 + 用量增值，匹配 AI 成本结构

## 2. 用户角色与权限

| 角色 | 说明 | 核心权限 |
|---|---|---|
| 超级管理员 | 平台运营方 | 机构管理、计费、系统配置 |
| 机构管理员 | 医院管理方 | 全局看板、团队管理、SOP/标签审核、考核数据 |
| 咨询师 | 一线业务人员 | 名下客户管理、录音上传、SOP 创建编辑、跟进执行 |

**数据隔离原则**：
- 咨询师仅可见名下客户；客户归属机构，支持一键转移（离职交接）
- 机构管理员可见本机构全部数据
- 多租户架构，机构间数据严格隔离

## 3. 功能需求

### 3.1 业务端（咨询师）

#### F1. 语音解析与客户标签自动生成 ⭐ 核心功能
- **输入**：面谈录音 MP3（单文件 ≤ 200MB，时长 ≤ 120 分钟，支持拖拽上传）
- **处理流水线**：
  1. ASR 语音转文字（支持说话人分离：咨询师 vs 客户）
  2. DeepSeek LLM 解析对话，提取结构化信息
  3. 自动生成客户多维度标签
  4. 自动生成/更新跟进策略
- **标签维度**（机构管理员可配置扩展）：
  | 维度 | 示例 |
  |---|---|
  | 基础信息 | 年龄段、性别、职业、地域 |
  | 消费能力 | 预算区间、价格敏感度、历史消费 |
  | 需求意向 | 关注项目（双眼皮/水光针/热玛吉…）、紧迫度、决策阶段 |
  | 顾虑点 | 怕痛、怕恢复期、价格顾虑、效果疑虑 |
  | 性格画像 | 决策风格（果断/犹豫）、沟通偏好 |
  | 复购潜力 | 关联项目推荐空间、生命周期阶段 |
- **关键要求**：
  - 每个标签可溯源至对话原句（点击标签 → 高亮跳转转写原文）
  - 标签可人工修正，修正留痕（用于 Prompt 迭代优化）
  - 异步任务 + 实时进度展示（解析中动画 → 完成通知）
- **UI 体验亮点**：解析过程展示"AI 分析流"动效（音频波形 → 文字流 → 标签逐个点亮），强化智能感

#### F2. 客户资产管理（智能 CRM）
- 客户 360° 档案：基础信息、标签云、消费记录、录音列表、跟进时间线
- 客户列表：按标签/意向度/阶段/最近跟进时间多维筛选排序
- 跟进时间线：聚合全部触点（录音解析、微信跟进、到店、成交）
- **UI 体验亮点**：客户详情页采用"全息档案"风格，标签以发光胶囊呈现，意向度用环形能量条展示

#### F3. 跟进策略自动生成与执行
- **策略生成**：客户标签 + 机构 SOP 模板 → AI 生成个性化跟进计划
  - 包含：跟进节奏（T+1/T+3/T+7）、每次触达目标、推荐话术
- **每日智能工作台**：
  - 今日待跟进客户数 + 优先级排序的客户卡片流
  - 每张卡片展示：跟进原因、当前阶段、AI 生成的个性化话术
  - 一键复制话术 → 人工发送微信（MVP 为半自动）
  - 执行状态标记：已完成 / 跳过（填原因）/ 延后
- **执行进度可视化**：每客户策略进度（已完成 N/M 触点）
- **UI 体验亮点**：工作台首屏为"任务指挥中心"，今日任务以 HUD 仪表盘呈现

#### F4. 跟进策略 SOP 管理
- 新建/编辑业务 SOP（如"水光针客户 90 天复购 SOP"）
- 支持修改 AI 生成的 SOP，修改后提交管理员审核方可启用
- SOP 编辑器：可视化节点流程编排（阶段 → 触点 → 话术模板）

### 3.2 管理端（机构管理员）

#### F5. 全局数据看板（指挥舱）
- 核心指标：客户总数、新增、跟进完成率、转化率、复购率、业绩金额
- 漏斗分析：线索 → 面谈 → 成交 → 复购
- 时间维度：日/周/月 + 环比同比
- **UI 体验亮点**：暗色"数据指挥舱"大屏风格，发光图表 + 数字滚动动效 + 实时数据流

#### F6. 咨询师团队管理
- 成员业务进度：名下客户数、今日跟进完成率、本月成交
- 考核数据：转化率、复购率、跟进及时率、录音上传率
- 客户归属一键转移（离职交接）

#### F7. 审核中心
- SOP 审核：通过/驳回 + 意见
- 标签与策略抽检：抽查 AI 输出是否符合机构标准
- 全部审核操作留痕

#### F8. 标准化配置
- 标签体系配置（维度、可选值）
- SOP 模板库（机构级标准模板）
- 话术风格配置（品牌语气、禁用词、医疗合规话术库）

## 4. UI/UX 设计规范 —— 未来科技感设计语言 ⭐

### 4.1 设计理念
**"医疗精密感 × AI 未来感"** —— 参考赛博 HUD、太空舱控制台、生物科技实验室的视觉语言，但保持医美行业的精致与信任感（避免过度游戏化）。

### 4.2 视觉系统

| 要素 | 规范 |
|---|---|
| 主题模式 | **暗色模式为默认**（深空蓝黑底 `#0A0E1A` / `#0D1326`），提供浅色模式切换 |
| 主色 | 电光青 `#00E5FF`（AI 智能元素专用色） |
| 辅助色 | 霓虹紫 `#7C4DFF`（渐变搭配）、能量绿 `#00FFA3`（成功/完成态） |
| 警示色 | 琥珀橙 `#FFB300`（待办）、警报红 `#FF4D6A`（逾期/风险） |
| 渐变 | 青→紫线性渐变用于按钮、进度条、关键数据高亮 |
| 玻璃拟态 | 卡片采用 Glassmorphism：`backdrop-blur` + 半透明背景 + 1px 发光描边 |
| 发光效果 | 关键元素带柔和 neon glow（`box-shadow` 青色光晕），hover 增强 |
| 字体 | 中文：思源黑体/系统字体；数字：等宽科技字体（如 JetBrains Mono / Orbitron 风格）用于数据展示 |
| 圆角 | 卡片 12px，按钮 8px，标签胶囊全圆角 |


### 4.3 动效规范
| 场景 | 动效 |
|---|---|
| 页面加载 | 骨架屏 + 内容渐入（fade-in + slide-up，200~300ms） |
| 数据看板 | 数字滚动递增（count-up）、图表绘制动画、实时数据流光效 |
| AI 解析中 | 音频波形律动 → 文字流式输出 → 标签逐个"点亮"（scale + glow） |
| AI 生成话术 | 打字机流式输出效果（对接 LLM streaming） |
| 任务完成 | 进度条能量填充 + 完成时粒子/光波扩散微动效 |
| 卡片交互 | hover 时轻微上浮（translateY -4px）+ 发光描边增强 |
| 页面切换 | 路由过渡动画（淡入淡出，避免过度炫技影响效率） |

**动效原则**：动效服务于"AI 正在工作"的感知反馈，单次动效 ≤ 400ms，提供"减弱动效"无障碍开关。

### 4.4 关键页面视觉定义
| 页面 | 视觉概念 |
|---|---|
| 登录页 | 深空背景 + 粒子流动 + 玻璃拟态登录卡片 + Logo 呼吸光效 |
| 咨询师工作台 | "任务指挥中心"：顶部 HUD 数据条 + 今日任务卡片流 + AI 助手悬浮入口 |
| 客户详情页 | "全息档案"：左侧客户信息 + 中部发光标签云 + 右侧时间线，意向度环形能量条 |
| 录音解析页 | 上传区为"能量舱"造型，解析中展示波形 + 转写流 + 标签实时点亮 |
| 管理看板 | "数据指挥舱"暗色大屏：发光折线/漏斗图 + 数字滚动 + 团队状态矩阵 |
| SOP 编辑器 | 节点流程图画布（类 Figma/n8n），节点间发光连线 |

## 5. 技术栈与架构 ⭐

### 5.1 开发模式
- **AI 辅助开发**：opencode + DeepSeek（chat 模型负责架构/编码，reasoner 模型负责复杂逻辑设计）
- 项目根目录维护 `AGENTS.md` 作为 AI 编码上下文（见配套文件）

### 5.2 前端技术栈
| 类别 | 选型 | 理由 |
|---|---|---|
| 框架 | **Next.js 14+（App Router）+ TypeScript** | 全栈一体、SSR、生态成熟，对 AI 代码生成友好 |
| UI 组件 | **shadcn/ui + Tailwind CSS** | 高度可定制，便于实现科技感设计系统；代码即组件，AI 生成质量高 |
| 动效 | **Framer Motion** | 声明式动画，覆盖全部动效规范 |
| 图表 | **ECharts**（暗色主题定制） | 发光图表、大屏看板能力强 |
| 状态管理 | **Zustand** + TanStack Query | 轻量；服务端状态缓存与乐观更新 |
| 表单 | React Hook Form + Zod | 类型安全校验 |
| 音频 | wavesurfer.js | 波形可视化 + 播放定位（标签溯源跳转） |
| 流程编排 | React Flow | SOP 可视化节点编辑器 |

### 5.3 后端技术栈
| 类别 | 选型 | 理由 |
|---|---|---|
| API 层 | **Next.js API Routes / Server Actions**（MVP）| 减少技术栈复杂度，单仓全栈 |
| 数据库 | **PostgreSQL + Prisma ORM** | 多租户、JSONB 存标签、类型安全 |
| 缓存/队列 | **Redis + BullMQ** | 录音解析异步任务队列、进度推送 |
| 对象存储 | S3 兼容存储（阿里云 OSS / MinIO） | 录音文件存储（加密） |
| 认证 | NextAuth.js（JWT + RBAC） | 三角色权限体系 |
| 实时推送 | SSE（Server-Sent Events） | 解析进度、AI 流式输出 |

### 5.4 AI 能力层
| 能力 | 选型 | 说明 |
|---|---|---|
| ASR 语音转写 | 阿里云 Paraformer / 腾讯云 ASR（POC 对比） | 需支持说话人分离，医美专业词热词表定制 |
| LLM 解析与生成 | **DeepSeek API**（chat + reasoner） | 标签提取、策略生成、话术生成；reasoner 用于复杂策略推理 |
| Prompt 管理 | 独立 `prompts/` 目录版本化管理 | 标签提取/策略生成/话术生成三套核心 Prompt |
| 输出约束 | JSON Schema 结构化输出 + Zod 二次校验 | 保证标签结构稳定可入库 |
| 合规过滤 | 医疗违禁词词库 + LLM 二次审查 | 话术生成后强制过滤 |

### 5.5 系统架构（MVP）
┌─────────────────────────────────────────────┐ │ Next.js 前端（暗色科技感 UI / SSE 订阅） │ └──────────────────┬──────────────────────────┘ │ ┌──────────────────▼──────────────────────────┐ │ Next.js API / Server Actions（RBAC 鉴权） │ ├──────────┬───────────────┬──────────────────┤ │ Postgres │ Redis+BullMQ │ OSS（录音存储） │ │ (Prisma) │ (异步任务队列) │ │ └──────────┴───────┬───────┴──────────────────┘ │ Worker 消费 ┌──────────▼───────────┐ │ 解析流水线 Pipeline │ │ ASR → DeepSeek 标签 │ │ → DeepSeek 策略生成 │ │ → 合规过滤 → 入库 │ └──────────────────────┘

### 5.6 部署
- MVP：Docker Compose 单机部署（App + Postgres + Redis + MinIO）
- 生产：云服务器 + 托管数据库，CI/CD 用 GitHub Actions

## 6. 非功能需求

| 类别 | 要求 |
|---|---|
| 合规安全 | 录音与客户数据加密存储；符合《个人信息保护法》；录音知情同意提示；AI 话术医疗广告合规过滤 |

| 数据隔离 | 多租户逻辑隔离（所有表带 `org_id`，Prisma 中间件强制过滤） |
| 审计 | 标签修改、SOP 审核、客户转移等敏感操作全留痕 |

## 10. 风险与开放问题

1. **录音合规**：客户知情同意流程需法务确认
2. **微信自动发送**：个微自动化有封号风险 → 优先企业微信 API
3. **ASR 选型**：医美专业词（项目名/药品名）识别准确率需 POC
4. **说话人分离**：多人面谈场景分离准确率需验证
5. **收费模式**：MVP 后通过试点机构访谈确定
6. **科技感 vs 效率**：动效需 A/B 验证不影响咨询师高频操作效率
